
# Kubernetes Hands-on Lab 01
## Installation, Cluster Setup, Pod Creation & Basic Pod Operations

### Objective
The objective of this lab was to set up a local Kubernetes environment and perform basic Kubernetes operations using `kubectl`. The tasks included verifying installations, starting a Kubernetes cluster, deploying an Nginx pod, inspecting the pod, forwarding ports, viewing logs, and accessing the pod interactively.

# Step 1: Verify kubectl Installation

Verify that kubectl is installed correctly.

```bash
kubectl version --client
```

Expected Output

```text
Client Version: v1.xx.x
```

Purpose

- Checks whether the Kubernetes CLI is installed.
- Displays the installed client version.

---

# Step 2: Verify Docker Installation

Minikube requires a container runtime. Docker is the recommended driver.

```bash
docker --version
```

Expected Output

```text
Docker version 28.x.x
```

Purpose

- Confirms Docker is installed.
- Ensures Minikube can use Docker as its driver.

---

# Step 3: Install Minikube

Initially, the following command returned:

```bash
minikube status
```

Output

```text
Command 'minikube' not found
```

This indicated that Minikube was not installed.

Install Minikube

```bash
curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64
```

```bash
sudo install minikube-linux-amd64 /usr/local/bin/minikube
```

Verify Installation

```bash
minikube version
```

Purpose

- Installs the local Kubernetes cluster manager.

---

# Step 4: Start Kubernetes Cluster

Start Minikube using Docker.

```bash
minikube start --driver=docker
```

Purpose

- Creates a single-node Kubernetes cluster.
- Starts the Kubernetes API Server.
- Starts etcd.
- Starts Scheduler.
- Starts Controller Manager.
- Starts Worker Node.

Verify Cluster

```bash
kubectl get nodes
```

Expected Output

```text
NAME        STATUS   ROLES           AGE
minikube    Ready    control-plane   2m
```

---


Purpose

Displays

# Step 6: Create an Nginx Pod

Command

```bash
kubectl run nginx --image=nginx
```

Purpose

Creates

- One Pod
- One Container
- Uses the official nginx image

```
kubectl
      │
      ▼
API Server
      │
Scheduler

Worker Node
      │
Container Runtime
      │
nginx Container
```

# Step 7: Verify Pod

```bash
kubectl get pods
```

Expected Output

```text
NAME      READY   STATUS    RESTARTS   AGE
nginx     1/1     Running   0          1m
```

Explanation

| Column | Meaning |
|---------|----------|
| READY | Running containers |
| STATUS | Current Pod status |
| RESTARTS | Restart count |
| AGE | Time since creation |

# Step 9: Port Forward

Command

```bash
kubectl port-forward pod/nginx 8080:80
```

Meaning

```
localhost:8080
        │
        ▼
Pod Port 80
```

Expected Output

```text
Forwarding from 127.0.0.1:8080 -> 80
Forwarding from [::1]:8080 -> 80
```

Important Notes

- This command **does not terminate automatically**.
- It waits for incoming HTTP requests.
- Keep this terminal open.
- Press **Ctrl + C** to stop port forwarding.

Open another terminal or browser:

```
http://localhost:8080
```

Expected Page

```
Welcome to nginx!
```

---

# Incorrect Port Forward Attempt

Executed

```bash
kubectl port-forward pod/nginx 8080:8080
```

Issue

The nginx container listens on **port 80**, not 8080.

Correct Command

```bash
kubectl port-forward pod/nginx 8080:80
```

# Step 11: Execute Commands Inside Pod

Command

```bash
kubectl exec -it nginx -- bash
```

If bash is unavailable

```bash
kubectl exec -it nginx -- sh
```

Purpose

Provides an interactive shell inside the running container.

Useful Commands

```bash
hostname
```

```bash
pwd
```

```bash
ls
```

Navigate to the nginx web directory

```bash
cd /usr/share/nginx/html
```

List files

```bash
ls
```

Display default web page

```bash
cat index.html
```

Exit

```bash
exit
```

# Troubleshooting Performed

## Issue 1

```
The connection to the server localhost:8080 was refused
```

Reason

- Kubernetes cluster was not running.

Resolution

- Installed Minikube.
- Started the cluster.
- Verified node status.

---

## Issue 2

```
Command 'minikube' not found
```

Reason

- Minikube was not installed.

Resolution

Installed Minikube manually.

---

## Issue 3

Port forwarding appeared to hang.

Reason

`kubectl port-forward` is designed to remain active until manually stopped.

Resolution

- Keep the terminal open.
- Access the application from another terminal or browser.
- Stop with **Ctrl + C** when finished.
